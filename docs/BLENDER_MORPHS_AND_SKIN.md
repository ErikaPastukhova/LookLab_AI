# Blender: shape keys и веса скина для манекена

Цель — чтобы диапазон слайдеров (ниже/выше «базы») соответствовал **осмысленным** формам, а не только «нейтраль → толще».

## Имена ключей (glTF / Three.js)

Сейчас в `male_advanced.glb` / `female_advanced.glb` в `mesh.extras.targetNames` (у тела на одну форму на примитив):

`arms`, `belly`, `chest`, `hips`, `legs`,
`arms_slim`, `belly_slim`, `chest_slim`, `hips_slim`, `legs_slim`

### Расширение A — «худой» полюс (рекомендуется)

Создайте отдельные Shape Keys от той же базовой геометрии. Если у вас уже есть `*_slim` (как сейчас в проекте), этот шаг можно пропустить.

| Зона (см. слайдер) | Ключ «толще» (уже есть) | Доп. ключ «худее» |
|--------------------|-------------------------|-------------------|
| Грудь              | `chest`                 | `chest_slim`      |
| Талия              | см. раздел B ниже     | `belly_slim` или `waist_slim` |
| Бёдра              | `hips`                  | `hips_slim`       |
| Руки               | `arms`                  | `arms_slim`       |
| Ноги               | `legs`                  | `legs_slim`       |

В Blender имена должны **точно** совпадать с таблицей (латиница, нижний регистр, подчёркивание).

Скрипт нормализует вес: у `base` см оба морфа = 0; при уходе к минимуму слайдера растёт `*_slim`, к максимуму — «толстый» ключ.

### Расширение B — талия отдельно от «живота»

- **`belly`** — визуально живот / перед торса (как сейчас).
- **`waist`** — сужение на уровне талии (бока/спина), без лишнего раздува всего передa.

Если в экспорте есть только `belly`, талия по-прежнему мапится на него.  
Если добавлен **`waist`**, обхват талии из UI идёт в **`waist`**, а не в `belly`.

Пара для slim:

- при использовании **`waist`** для «толще» — желательно **`waist_slim`** для «худее»;
- иначе — **`belly_slim`**.

## Проверка скина (артефакты при сильных морфах)

1. **Weight Paint** на зонах с большим сдвигом вершин (подмышки, шея, пах, колени): нет ли вершин с весом только на одну кость там, где нужен плавный переход.
2. **Normalize all** в режиме весов, затем прогнать экстремальные позы + Shape Key на 1.0.
3. В glTF: убедиться, что у тела **один** skinned mesh с морфами (как сейчас), `JOINTS_0` / `WEIGHTS_0` не обнулились после добавления ключей.
4. После экспорта откройте консоль браузера: при загрузке модели выводится строка **`[Mannequin] morph targets:`** — сверьте список с ожидаемыми именами.

## Экспорт из Blender

- glTF 2.0, отметить **Shape Keys**, проверить что в JSON попали `extras.targetNames` (делает официальный экспортёр при корректных именах меша/ключей).

## Зависимость от имён костей (теплокарта)

Цветовая разметка в `updateColors()` опирается на подстроки имён костей в `bonesList`:

- грудь: `spine_03` или `clavicle` или `neck` или `chest`
- талия: `spine_01` или `spine_02`
- бедра: `pelvis` или `root` или `hip`
- руки: `upperarm` или `lowerarm` или `hand`
- ноги: `thigh` или `calf` или `leg` или `foot`

При реворке rig/armature важно сохранить эти паттерны, иначе heatmap перестанет соответствовать зоне.

## Расширение C — `weight_fullness` и `figure_*` (коррективы)

В `male_advanced.glb` / `female_advanced.glb` добавлены дополнительные shape keys:

- глобальная полнота:
  - `weight_fullness`
  - `weight_fullness_slim`
- пресеты типа фигуры (распределение массы и “талия/груша/перевёрнутый”):
  - `figure_rectangle` / `figure_rectangle_slim`
  - `figure_hourglass` / `figure_hourglass_slim`
  - `figure_pear` / `figure_pear_slim`
  - `figure_inverted` / `figure_inverted_slim`

В веб‑коде они применяются как корректирующие morph targets поверх мерок (chest/waist/hips/arms/legs).
